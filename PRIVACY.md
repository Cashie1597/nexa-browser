# Privacy

Nexa Browser is designed to be privacy-first by default.

The goal is to give users a clean browsing experience with strong privacy controls, minimal tracking, and clear settings.

## Privacy Goals

- No unnecessary telemetry
- No selling user data
- No hidden tracking
- Clear privacy settings
- Private browsing mode
- Per-site privacy controls
- Built-in content blocking
- Local-first storage where possible

## Planned Privacy Features

### Shield Controls

A browser shield button will show the current privacy status for each site.

Planned shield options:

- Block trackers
- Block ads
- Block popups
- Force HTTPS where possible
- Disable scripts per site
- Clear site data
- Manage cookies

### Content Blocking

Nexa will support built-in content blocking rules for ads, trackers, and known unwanted scripts.

Planned blocker types:

- Tracker blocker
- Ad blocker
- Cookie banner blocker
- Popup blocker
- Social widget blocker

### Private Browsing

Private browsing mode should:

- Avoid saving history
- Avoid saving cookies after the session ends
- Avoid saving form data
- Avoid saving AI conversation context unless the user chooses to save it

### Per-Site Settings

Users should be able to manage privacy settings for individual websites.

Examples:

- Allow cookies on this site
- Disable blocker on this site
- Always open in reader mode
- Always request desktop site
- Clear data for this site

## AI Privacy

AI features should be clear and user-controlled.

Nexa should not send page content to an AI provider unless the user chooses an AI action, such as:

- Summarise this page
- Explain selected text
- Translate selected text
- Ask AI about this page

## Local Data

Where possible, Nexa should store user data locally on-device.

Local data may include:

- Bookmarks
- History
- Tabs
- Settings
- Extension states
- Saved prompts
- Page notes

## Privacy Promise

Nexa should be built around a simple rule:

> The browser should work for the user, not against the user.
