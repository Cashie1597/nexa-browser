# AI Assistant

Nexa Browser includes an AI assistant designed to help users understand, summarise, translate, and work with webpages.

The AI assistant should feel built into the browser rather than added on top.

## AI Goals

- Help users understand webpages faster
- Summarise long articles
- Explain selected text
- Translate selected text
- Rewrite selected text
- Answer questions about the current page
- Turn webpages into notes
- Keep the user in control of what is sent to AI

## Planned AI Features

### Page Summary

Summarise the current webpage into:

- Key points
- Short summary
- Detailed summary
- Action items
- Important links

### Ask About Page

Users should be able to ask questions about the current page.

Examples:

- What is this page about?
- Summarise this in simple terms.
- What are the key risks?
- What should I do next?
- Turn this into notes.

### Selected Text Actions

When a user selects text, Nexa should offer AI actions:

- Explain
- Summarise
- Translate
- Rewrite
- Make shorter
- Make clearer
- Turn into bullet points

### AI Sidebar

The AI sidebar should work especially well on iPad.

Planned layout:

- Browser on the left
- AI assistant on the right
- Current page context available when the user allows it

### Prompt Shortcuts

Nexa should include quick AI commands such as:

- Summarise page
- Explain like I am new to this
- Extract steps
- Find important details
- Rewrite professionally
- Translate this
- Create study notes

## AI Providers

Planned provider support:

- OpenAI
- OpenRouter

Future possible support:

- Local models
- Custom API endpoints

## AI Privacy

Nexa should not automatically send webpage content to an AI provider.

The user should choose when to use AI actions.

Examples of user-triggered AI actions:

- Summarise this page
- Explain selected text
- Ask AI about this page
- Translate this text

## First AI Feature to Build

The first AI feature should be:

> Summarise the current page using the page title, URL, and extracted readable text.
